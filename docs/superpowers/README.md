# Design And Implementation Records

The dated files under `specs/` and `plans/` are historical engineering
records. They preserve the design decisions and task sequences used to build
the repository; checkbox state in an implementation plan is not a live
project-status tracker.

Release status is defined by the root `README.md`, Git tags, merged pull
requests, and their required CI results. M0-M9 were completed for `v1.0.0`.
Later maintenance work is recorded by subsequent commits and releases.

New work should use a new dated specification and plan instead of rewriting
the execution state of an older record.
